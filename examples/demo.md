# Demo Session

Full terminal session demonstrating `cvmfs-debug-rescue` extracting a
stacktrace from a D-state process.

Environment: Arch Linux, kernel 6.x, GDB 17.1

## Setup


```

$mkdir -p /tmp/testmount$ ./toy_fuse -f /tmp/testmount &
[1] 8769

$ cat /tmp/testmount/hello
hello world

```

## Triggering D-State


```

$ stat /tmp/testmount/hang &
[2] 8789

$ kill -9 8789

$ ps -o pid,stat,wchan,cmd -p 8789
PID STAT WCHAN  CMD
8789 D    reques stat /tmp/testmount/hang

```

The process is now in D-state. It cannot be killed or debugged.

GDB auto-downloaded debug symbols via debuginfod, resolved the ASLR
addresses, and identified the exact source line where the process froze:
`statx.c:28` — the glibc wrapper for the `statx` system call.

Register values confirm the syscall arguments:
- `RDI = 0xffffff9c` = `AT_FDCWD` (-100) — relative to current directory
- `RSI = 0x7ffe1fecdc35` = pointer to `"/tmp/testmount/hang"`
- `RDX = 0x900` = flags (`AT_SYMLINK_NOFOLLOW | AT_STATX_SYNC_AS_STAT`)
- `R10 = 0xfff` = mask (`STATX_ALL`)
- `R8 = 0x7ffe1fecd310` = pointer to the `statx` result buffer
